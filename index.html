เครื่องมืออัปเดตจำนวนสินค้า vRich จากไฟล์ Excel

ประมวลผลในเบราว์เซอร์ของผู้ใช้เท่านั้น ไม่ส่งไฟล์ขึ้นเซิร์ฟเวอร์

ทำงานในเครื่องผู้ใช้เท่านั้น

เว็บนี้ใช้ทำอะไร

เว็บนี้ใช้สร้างไฟล์ vrich_import_update_qty.xlsx สำหรับอัปเดตจำนวนสินค้าเข้า vRich โดยอ่านข้อมูลจากไฟล์ Excel เต็ม 3 ไฟล์ที่ผู้ใช้เลือกเอง ระบบจะใช้ข้อมูลทั้งแถวจากไฟล์ vRich และเปลี่ยนเฉพาะคอลัมน์ จำนวน ด้วยจำนวนจากไฟล์ JST เฉพาะรหัสที่อยู่ใน sold_today เท่านั้น

ระบบไม่แก้ไฟล์ต้นฉบับ ไม่ import เข้า vRich เอง ไม่ใช้ API ไม่ดึงเว็บ และไม่อัปโหลดไฟล์ไปที่ใด หลังประมวลผลแล้วต้องตรวจ summary_report.xlsx และ report อื่นก่อนนำไฟล์ไป import ด้วยตัวเอง

ไฟล์ที่ต้องเตรียม

  • stock_vrich619.xlsx คือไฟล์ master จาก vRich ใช้เป็นฟอร์มหลักและข้อมูลสินค้าทั้งแถว
  • Item jsterp.xlsx คือไฟล์จาก JST ใช้เฉพาะจำนวนที่แม่นกว่า
  • sold_today.xlsx คือไฟล์รหัสสินค้าที่ขายวันนั้น หรือรหัสที่ต้องการอัปเดต

คำเตือนสำคัญ

  • ห้ามนำไฟล์ Excel จริงของบริษัทขึ้น GitHub
  • ห้าม import ถ้าสถานะเป็น FAIL หรือ FAIL_DUPLICATE
  • ถ้าเป็น PASS_WITH_EXCLUSION ต้องมั่นใจว่ารหัสที่ข้ามถูกต้อง
  • ถ้ารหัสใดไม่มีใน JST จริง สามารถเลือกข้ามรายการได้ แต่ผู้ใช้ต้องตรวจสอบและยืนยันเองก่อน

ข้อจำกัดเรื่องไฟล์ใหญ่

  • เว็บนี้อ่านไฟล์ Excel ใน browser ของผู้ใช้ จึงใช้หน่วยความจำของเครื่องผู้ใช้โดยตรง
  • ไฟล์ที่ใหญ่มาก หรือมีรูปภาพ/format จำนวนมาก อาจทำให้ browser อ่านไม่ไหว
  • ถ้าไฟล์ใหญ่กว่า 30 MB ระบบจะแสดงคำเตือนก่อนประมวลผล
  • ถ้าไฟล์ใหญ่กว่า 80 MB ระบบจะ block และแนะนำให้ใช้ไฟล์ export แบบไม่มีรูปภาพ หรือใช้เวอร์ชัน Python
  • อนาคตอาจเพิ่มทางเลือก CSV/TSV สำหรับไฟล์ใหญ่ แต่เวอร์ชันนี้ยังประมวลผล Excel เท่านั้น

ขั้นตอนใช้งานแบบละเอียด

  1. โหลดไฟล์จำนวนสินค้าจาก vRich แล้วเตรียมเป็น stock_vrich619.xlsx
  2. โหลดไฟล์จำนวนสินค้าจาก JST แล้วเตรียมเป็น Item jsterp.xlsx
  3. เตรียม sold_today.xlsx ให้มีคอลัมน์รหัสสินค้า
  4. อัปโหลดไฟล์ทั้ง 3 ช่องด้านล่าง
  5. กด ตรวจสอบไฟล์ เพื่อดูชื่อไฟล์ ชื่อชีต จำนวนแถว และคอลัมน์ที่พบ
  6. กด ประมวลผล
  7. ตรวจรายการที่ไม่พบและรายการซ้ำให้ครบ
  8. ถ้ามีรหัสที่ไม่มีใน JST และยืนยันแล้วว่าจะไม่อัปเดตรหัสนั้น ให้เลือกข้ามรายการแล้วกดยืนยัน
  9. ดาวน์โหลดไฟล์ผลลัพธ์และ report ทั้งหมด
  10. เปิด summary_report.xlsx เพื่อตรวจสถานะ จำนวนรายการ และรหัสที่ถูกข้าม
  11. ค่อยนำ vrich_import_update_qty.xlsx ไป import เข้า vRich ด้วยตัวเอง
